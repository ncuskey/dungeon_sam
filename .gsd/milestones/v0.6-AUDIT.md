## Concerns
- **Hardcoded Debug Flag**: `debugNoEnemies` is currently defaulted to `true` in `gameStore.ts`. This is great for testing doors but must be disabled or moved to a real debug menu before a "real" release.
- **Door Mesh Thickness**: Doors appear very thin (0.1 units). While historically accurate to sprite-based games, they can look like "paper" from certain angles.

## Recommendations
1. Implement a proper **Debug UI** (toggleable) to control enemy spawning and other dev flags.
2. Consider adding bit of depth to door meshes if visual immersion becomes a priority.
3. Consolidate door math into a utility function to avoid duplication if more interactive objects are added.

## Technical Debt to Address
- [ ] Move `debugNoEnemies` to a persistent setting or runtime toggle.
- [ ] Refactor `LevelRenderer` door mapping logic for better readability.
